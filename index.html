DAM Admin User Guide

Guide complet pour ajouter et rendre visibles des assets dans le Media Center, du chargement dans le back-end à l’affichage sur le front-end.

Back-end vs front-end

Le back-end est l’espace d’administration du DAM : c’est là qu’on charge les assets, complète les métadonnées, crée les filtres et organise les Libraries.

Le front-end est l’interface visible par les utilisateurs : c’est le Media Center où les assets sont consultés, recherchés et téléchargés.

Key principle

Un asset uploadé dans le DAM n’est pas automatiquement visible dans le Media Center.

Il doit être connecté au front-end via une Library, puis affiché sur une page avec une Asset Gallery.

Ajouter un ou plusieurs fichiers dans le bon dossier du DAM et compléter leurs informations.

  • 1 Aller dans Explore.
  • 2 Sélectionner le dossier dans lequel les assets doivent être chargés.
  • 3 Cliquer sur Upload asset et importer les fichiers souhaités.
  • 4 Compléter les métadonnées nécessaires.
  • 5 Ajouter les tags appropriés.
  • 6 Cliquer sur Save and Finish.
À noter

Les tags et métadonnées sont essentiels : ils conditionnent le bon fonctionnement des filtres créés à l’étape suivante.

Créer un filtre de recherche enregistré, qui permettra d’alimenter automatiquement une Library avec les bons assets.

  • 1 Dans le back-end, cliquer sur Search.
  • 2 Saisir le nom exact du document ou de l’asset dans la barre de recherche.
  • 3 Vérifier que les bons résultats apparaissent.
  • 4 Cliquer sur l’icône Save icône disquette.
  • 5 Sélectionner Save as.
  • 6 Donner un nom clair au filtre. Ex : Care & Wellness pictures
  • 7 Enregistrer le filtre.
Convention de nommage

Utiliser un nom descriptif et cohérent avec la future Library. Ex : Care & Wellness pictures pour le filtre → Care & Wellness pour la Library.

Créer une Library liée au filtre créé précédemment, afin de rendre les assets disponibles dans le Media Center.

  • 1 Dans le menu de gauche, cliquer sur Libraries.
  • 2 Cliquer sur New.
  • 3 Renseigner le nom dans le champ Library name. Ex : Care & Wellness
  • 4 Sélectionner symrise.celum.cloud.
  • 5 Cliquer sur Assets dans le menu de gauche.
  • 6 Cliquer sur Add a filter.
  • 7 Sélectionner le filtre créé à l’étape précédente.
  • 8 Cliquer sur Create en haut à droite.

Rendre la Library disponible dans le portail Media Center, visible des utilisateurs.

  • 1 Aller dans Experience.
  • 2 Cliquer sur les trois petits points en haut à droite.
  • 3 Cliquer sur Edit Portal.
  • 4 Aller dans l’onglet Libraries.
  • 5 Cliquer sur Add libraries et sélectionner la Library créée.
  • 6 Cliquer sur Save.

Ajouter une galerie d’assets sur la page souhaitée du Media Center via un widget Asset Gallery.

  • 1 Aller sur la page à mettre à jour.
  • 2 Sous le titre, cliquer sur + Add section.
  • 3 Dans Choose layout, sélectionner une mise en page en une colonne.
  • 4 Cliquer sur + Add widget.
  • 5 Sélectionner le widget Asset Gallery.
  • 6 Dans le panneau de gauche, aller dans Library et sélectionner la Library créée.
  • 7 Vérifier que le contenu apparaît correctement sur la page.
  • 8 Cliquer sur Update en bas à gauche.
Vérification

Toujours contrôler l’affichage côté utilisateur après avoir ajouté une nouvelle Library.

Checklist de dépannage

À vérifier si le contenu n’apparaît pas dans le Media Center.

L’asset a bien été uploadé dans le bon dossier.
Les métadonnées et les tags sont bien renseignés.
Le filtre contient bien les bons assets.
La Library est bien reliée au filtre.
La Library a bien été ajoutée dans le portail.
Le widget Asset Gallery utilise la bonne Library.

Bonnes pratiques

Quelques réflexes simples pour garder un Media Center propre et fiable.

Nommage cohérent

Garder une logique entre le filtre et la Library. Ex : Care & Wellness pictures → Care & Wellness.

Tags avant publication

Compléter métadonnées et tags avant de créer le filtre, pour éviter les assets manquants.

Contrôle front-end

Vérifier systématiquement l’affichage côté utilisateur après toute modification.

Une Library par thème

Créer des Libraries ciblées plutôt qu’une seule grande Library globale.

Résumé rapide

Upload → Metadata & tags → Filter → Library → Front-end → Asset Gallery
